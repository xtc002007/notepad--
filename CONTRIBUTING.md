# 贡献指南

感谢你关注本项目！请遵循以下步骤参与开发：

## 参与步骤

1. **Fork 本仓库**：点击右上角的 Fork 按钮。

2. **克隆到本地**：
   ```bash
   git clone https://github.com/<你的用户名>/notepad--.git
   cd notepad--
   ```

3. **创建特性分支**：请基于 `main` 分支创建一个新分支。
   ```bash
   git checkout -b feature/你的功能名
   # 或
   git checkout -b fix/修复的问题
   ```

4. **提交更改**：
   - 保持提交信息（Commit Message）清晰简洁
   - 建议使用以下格式：
     - `feat: 添加新功能`
     - `fix: 修复某个问题`
     - `docs: 更新文档`
     - `style: 代码格式调整`
     - `refactor: 代码重构`

5. **发起 Pull Request (PR)**：
   - 提交前请确保本地测试通过
   - 将分支推送到你的 Fork 仓库：
     ```bash
     git push origin feature/你的功能名
     ```
   - 在 GitHub 上点击 "Compare & pull request"

6. **等待审核**：
   - 维护者会在 48 小时内进行 Code Review
   - 如有需要修改的地方，请及时更新

## 代码规范

- 保持代码风格一致
- 添加必要的注释
- 确保代码通过 TypeScript 类型检查
- 遵循项目现有的代码结构

## 测试

在提交 PR 之前，请确保：
- 代码能够正常编译
- 现有功能没有被破坏
- 新功能经过充分测试

## 问题反馈

如果你发现了 bug 或有功能建议，欢迎：
- 提交 Issue 详细描述问题
- 在 Issue 中讨论解决方案
- 提交 PR 来修复问题

## 行为准则

- 尊重所有贡献者
- 保持友好和专业的交流
- 接受建设性的批评和建议
